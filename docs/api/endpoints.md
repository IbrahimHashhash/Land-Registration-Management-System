# API Reference

Base URL: `http://localhost:8000` (configurable via the frontend `VITE_API_URL`).
Interactive docs: `http://localhost:8000/docs` (Swagger UI auto-generated by FastAPI).

Endpoints that act on staff actions require an `X-Staff-Role` header with
value `surveyor` or `registrar`.

---

## Applicants

### `POST /applicants/`
Create an applicant profile. Request body:
```json
{
  "full_name": "Nour Ahmad",
  "applicant_type": "citizen",
  "identity": { "national_id": "990000001", "verified": false, "verification_method": "otp_stub" },
  "contacts": { "email": "nour@example.com", "phone": "+97059..." },
  "address":  { "city": "Ramallah", "neighborhood": "Al Tireh", "zone_id": "ZONE-RM-01" }
}
```
Returns `201 Created` with the applicant. `409 Conflict` if `national_id` already exists.

### `GET /applicants/by-national-id/{national_id}`
Lookup an applicant by national ID. Used by the applicant portal sign-in flow.
`404 Not Found` if no applicant exists with that ID.

### `GET /applicants/{applicant_id}`
Public profile for the applicant.

### `GET /applicants/{applicant_id}/applications`
All applications submitted by this applicant (sorted by submission date desc).

### `GET /applicants/{applicant_id}/objections`
All objections filed by this applicant.

## Land Applications

### `POST /applications/`
Create a new land application. Optional `Idempotency-Key` request header â€” sending the same key twice returns the original record.
```json
{
  "application_type": "ownership_transfer",
  "priority": "normal",
  "applicant_ref": { "applicant_id": "APP-XXXXXXXX", "applicant_type": "citizen" },
  "parcel": {
    "parcel_no": "145", "block_no": "12", "basin_no": "3", "zone_id": "ZONE-RM-01",
    "geometry": { "type": "Polygon", "coordinates": [[[35.2001, 31.9021], ...]] }
  },
  "description": "Ownership transfer for parcel 145/12"
}
```
Application starts in `submitted`.

### `GET /applications/`
List with filtering, sorting, pagination.

| Query param        | Description |
|--------------------|-------------|
| `status`           | Exact status, or `All Statuses` |
| `application_type` | Exact type, or `All Types` |
| `zone_id`          | Exact zone, or `All Zones` |
| `search`           | Regex match against `application_id`, `applicant_ref.applicant_id`, `parcel.parcel_no` |
| `sort_by`          | Field to sort by (default `submission_date`) |
| `order`            | `asc` or `desc` (default `desc`) |
| `page`             | 1-based page (default 1) |
| `page_size`        | 1â€“100 (default 10) |

Response: `{ items, total, page, page_size, total_pages }`.

### `GET /applications/{application_id}`
Full application document.

### `PATCH /applications/{application_id}/transition`
Move the application to another state. Body:
```json
{ "to_state": "pre_checked", "note": "optional", "reason": "required if to_state=rejected", "actor_id": "staff_14" }
```
Enforces the [state machine](../workflow/state-machine.md) and the rule
checks (legal_review needs uploaded docs, surveyed needs report, etc.).

### `POST /applications/{application_id}/hold`
`{ "reason": "...", "actor_id": "staff_14" }`. Reason is required.

### `POST /applications/{application_id}/reject`
`{ "reason": "...", "actor_id": "staff_14" }`. Reason is required.

### `POST /applications/{application_id}/certificate`
Generate certificate metadata. Application must be in `approved`.
```json
{ "certificate_type": "ownership_certificate", "full_name": "Nour Ahmad", "issued_by": "registrar_09" }
```
Application moves to `certificate_issued`. Notifies the applicant.

## Documents, Comments, Objections, Timeline

### `POST /applications/{application_id}/documents`
Upload document metadata.
```json
{ "document_type": "ownership_deed", "file_name": "deed.pdf", "file_path": "/uploads/deed.pdf" }
```

### `GET /applications/{application_id}/documents`
List all documents for the application.

### `PATCH /applications/{application_id}/documents/{document_id}/verify` _(staff)_
Set the verification status of a document.
```json
{ "verification_status": "verified", "by": "<registrar_id>" }
```
`verification_status` is one of `verified | pending_review | rejected`.

### `POST /applications/{application_id}/comments`
`{ "author_id": "APP-XXXXXXXX", "text": "..." }`

### `POST /applications/{application_id}/objections`
`{ "author_id": "APP-XXXXXXXX", "reason": "...", "supporting_documents": [] }`.
Forces the application to `under_objection`.

### `GET /applications/{application_id}/timeline`
Returns the `performance_logs.event_stream` for the application, sorted by time.

## Staff (`X-Staff-Role` required)

### `POST /staff/`
Create a surveyor or registrar.

### `GET /staff/?role=surveyor`
List staff, optionally filtered by role.

### `GET /staff/{staff_id}`
Staff profile with computed performance summary.

### `GET /staff/{staff_id}/tasks`
Tasks assigned to a surveyor, enriched with `parcel_number`, `zone`, and `priority`.

### `POST /applications/{application_id}/auto-assign-surveyor`
Apply the assignment policy (zone match + availability + workload + skill +
priority) and create a survey task. Application must be in `survey_required`.

### `PATCH /applications/{application_id}/reassign-surveyor`
`{ "new_surveyor_id": "<staff_id>" }`. Updates the survey task and reduces/raises workload.

### `PATCH /applications/{application_id}/survey-milestone`
`{ "type": "<milestone>", "by": "<surveyor_id>", "meta": {} }`.
Allowed `type` values: `visit_scheduled | arrived_on_site | survey_started | survey_completed`.
Monotonic â€” earlier milestones are rejected with 400.

### `POST /applications/{application_id}/field-note`
`{ "note": "...", "by": "<surveyor_id>" }`.

### `POST /applications/{application_id}/survey-report`
`{ "report_title": "...", "findings": "...", "uploaded_by": "<surveyor_id>" }`.
Moves the application to `surveyed` and the task to `report_uploaded`.

### `PATCH /applications/{application_id}/registrar-review`
```json
{ "decision": "approved", "reviewed_by": "<registrar_id>", "notes": "...", "rejection_reason": "..." }
```
`decision` is one of `approved | rejected | on_hold | missing_documents | under_objection`.
Enforces `ALLOWED_NEXT` â€” the registrar cannot skip workflow steps.

## Errors

The API uses standard FastAPI / HTTPException JSON error bodies:
```json
{ "detail": "Cannot move from pre_checked to approved" }
```

Common codes:
- `400` â€” validation or workflow rule violation
- `403` â€” missing or invalid `X-Staff-Role` for staff endpoints
- `404` â€” resource not found
- `409` â€” duplicate (e.g. `national_id`)
- `422` â€” request body fails Pydantic validation
